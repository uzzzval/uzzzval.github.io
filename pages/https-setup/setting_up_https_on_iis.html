How to configure your IIS server for HTTPS

- Ujjawal Sharma

1. Open the Intenet Information Services(IIS) and you will observe a setup like as shown below, if you have already set up a default HTTP server. If not click here to do so. On the right panel, click on Bindings.




2. Now, you will be able to see the below screen. Click on Add.




3. You will now be prompted to fill in the up some details as shown in the following screenshot.




4. Select Type as https ,leave IP address as All Unassigned, change Port Number to 443. Fill in Host name to the name of your domain for which you created the certificate. Select the SSL Certificate from the dropdown. The dropdown contains list of all the Certificates which you will add to your server. Click on OK.




5. You will now observe your filled in details as shown below. Click Close.




6. You will now observe a new entry on the right side panel under Browser Website section. Try clicking on it.




7. You will now observe that it will give you an error saying that the site cannot be reached. We will now move towards the next steps.




8. On the Explorer on the Windows Operating System, navigate to C:\Windows\System32\drivers\etc and try editing hosts file.




9. But before editing, you have to get the proper permissions to edit the file. Right click on hosts file and click on properties.




10. On the properties box, navigate to Security tab, then under the Group or user names section, select Users, then click Edit. You will now observe the Permission for hosts box. Select Users, click on Add. Now, you will observe Select Users or Groups box. Under the Enter the object names to select, enter the name of Computer's username and click on Check Name.




11. After you click Check Name in the previous step, the entry under Enter the object name to select should turn to as shown following. Click OK.




12. You will now observe the Permissions for hosts box changed to as shown below. Click Apply. Click Ok. Do the same for the properties box as well.




13. Open the hosts file in a notepad. Add the entry for your site and point it to 127.0.0.1.




14. Try reaching out the site again. This time you will observe that the browser will give you Unsecure Connection error, as shown below. We will now perform the next steps.




15. From the windows search option, type mmc, when the mmc icon appears, right click on it and then click on Run as administrator.




16. Under the Console Root tree, expand Certificates (Local Computer), then expand Personal, then Certificates. Now, you will observe your site certificate in the list. Right Click and copy it.




17. Now, expand the Trusted Root Certificates option and click on Certificates. Paste the copied certificate here.




18. You will observe your copied certificate in the Trusted Root Certificates list. Close the mmc window.




19. On the IIS server application window, click on Bindings again, and then remove the http Site Bindings options, so that your website should be accessible only over the secured layer.




20. After you have removed it, you will observe that it disappear from the right pane Browser website section as well. We are all done!




21. Try clearing the Broswer's cache (using ctrl+alt+del) and then hit the website again. You will now observe your website is getting delivered to you on a secure channel.
